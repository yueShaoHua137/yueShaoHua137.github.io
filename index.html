关于首次创建个人博客所遇到的问题及解决方法

发表于 2025-06-26

关于首次创建个人博客所遇到的问题及解决方法

发表于 2025-06-26

安装Node.js，git，hexo以及检查安装

  1. 进入网站Node.js安装网站

node下载网站链接

分别点击红圈按键，下载Node.js。

  1. 安装git

git下载链接

同样按顺序点击红圈按钮。
需要根据不同的操作系统选择所需的下载，我这里选择的是Windows版。

  1. 安装hexo

按win+R键，打开cmd，进入事先创建好的blog文件夹，输入npm install -g hexo-cli，自动下载hexo。

命令：npm install -g hexo-cli

  1. 检查

同样在cmd中分别输入下列命令，检查安装。出现版本信息说明安装成功。

命令：hexo -v

命令：git -v
命令：node -v
命令：npm -v

创建仓库

进入GitHub网站，注册账号。
点击Create repository。

注意：仓库名字必须和账号名字相同！！后面加.github.io。

配置密钥

桌面右键点击git bash。
代码：ssh-keygen -t rsa -C “xie13718800562@163.com“
（引号内填写自己的邮箱，我这里填写的是自己的）
然后点4下回车。

然后进入c盘，用户，管理员文件夹，进入.ssh文件夹，打开以.pub为后缀的文件。
打开后直接复制所有内容。

再次打开GitHub，选择settings中的ssh and gpg keys。
点击新建一个ssh keys。

将刚刚全选复制的代码内容粘贴进入。之后创建。

回到桌面，右键git bash。
输入命令验证。

命令：ssh -T git@github.com
回车。

输入：yes
回车。

！！问题！！
这里前两次是出现了一个ssh: connect to host github.com port 22: connection refused的报错。
是因为我使用了加速器，关闭后即可验证成功。

本地部署

初始化

在创建好的blog文件夹内右键git bash。
输入命令进行初始化。

命令：hexo init

命令：npx hexo install
命令：hexo g
命令：hexo s
生成并保存。

打开所给链接 hexo博客

回到输入框，按CTRL+c停止。

！！问题！！

在hexo s时遇到的端口占用问题。

问题如图。我先是根据csnd中所查询到的办法，输入命令netstat -aon|findstr 4000，试图查找到占用4000端口的程序并结束它，但在使用后发现依旧不能正常运行。

所以我只能再尝试第二种方法。
在_config.yml内添加下列代码，更改hexo s运行时默认的端口号。
server:
port: 4001
compress: true
header: true

结果就是可以正常运行了。

上线

打开blog文件夹中的_config文件，做出如下修改。

根据个人需要，更改title，subtitle，author，language和时区等等信息。

更改repository，输入本人GitHub仓库的链接，然后保存退出。

在文件夹里git bash，分别输入以下命令。

命令：hexo g
命令：hexo d
生成并上传。

配置

输入命令。

命令：git config –global user.email “xie13718800562@163.com“
命令：git config –global user.name “yueShaoHua137”
具体邮箱和名字改为自己的即可。

然后再一次上传，会弹出一个登录窗口。

然后就可以输入链接进入自己的博客了！
